### 1. Текущая погода по координатам
Возвращает данные о температуре, скорости ветра и давлении на момент запроса.

* **URL:** `/forecast`
* **Метод:** `GET`
* **Параметры запроса:**
    * `lat` (float): Широта
    * `lon` (float): Долгота
* **Пример ответа:**
    ```json
    {
        "temperature": 18.5,
        "wind_speed": 3.4,
        "pressure": 1012.1
    }
    ```

---

### 2. Регистрация пользователя
Создает нового пользователя и возвращает его ID.

* **URL:** `/users`
* **Метод:** `POST`
* **Тело запроса:**
    ```json
    { "user_name": "string" }
    ```
* **Пример ответа (201):**
    ```json
    { "user_id": 1 }
    ```

---

### 3. Добавление города для пользователя
Добавляет город в список отслеживаемых для конкретного пользователя и инициирует загрузку прогноза на текущий день.

* **URL:** `/users/{user_id}/cities`
* **Метод:** `POST`
* **Тело запроса:**
    ```json
    {
        "city_name": "Gorno-Altaisk",
        "latitude": 51.95,
        "longitude": 85.96
    }
    ```
* **Пример ответа:**
    ```json
    { "city_id": 5 }
    ```

---

### 4. Список городов пользователя
Возвращает список названий всех городов, добавленных пользователем.

* **URL:** `/users/{user_id}/cities`
* **Метод:** `GET`
* **Пример ответа:**
    ```json
    ["Gorno-Altaisk", "Moscow", "Novosibirsk"]
    ```

---

### 5. Прогноз по времени
Возвращает погоду из базы данных на указанное время с возможностью фильтрации параметров. Недопустимые параметры игнорируются.

* **URL:** `/users/{user_id}/cities/{city_name}/forecast`
* **Метод:** `GET`
* **Параметры запроса:**
    * `time` (string): Время в формате `HH:MM` (например, `12:00`).
    * `params` (string, optional): Список полей через запятую. Доступны: `temperature`, `humidity`, `wind_speed`, `precipitation`.
* **Пример запроса:** `/users/1/cities/Moscow/forecast?time=15:00&params=temperature,humidity`
* **Пример ответа:**
    ```json
    {
        "temperature": -10.5,
        "humidity": 82
    }
    ```

---

## Тесты

Тесты покрывают репозитории (работа с SQLite), сервисный слой (бизнес-логика: регистрация, привязка городов, разграничение доступа между пользователями, фильтрация параметров прогноза) и HTTP-эндпоинты целиком (через `httpx.ASGITransport`, без поднятия реального сервера). Обращения к open-meteo подменяются фейковым HTTP-клиентом, поэтому тесты детерминированы и не требуют сети.


### Запуск тестов

```bash
pytest
```

# Усложнения

1. Реализована возможность запустить проект с помощью docker-compose без необходимости ручной настройки окружения.

Проект запустится только в том случае, если все тесты проходят проверку.

*Для локального запуска*
```bash
docker-compose -f docker-compose.local.yml up --build
```

*Для запуска на сервере*
```bash
docker-compose -f docker-compose.vps.yml up --build
```

2. Проект развернут на моем VPS сервере с помощью nginx. API доступно по адресу kirtrub.ru

Пример запроса:
```
curl https://kirtrub.ru/forecast?lon=70&lat=64
```
Ответ:
```
{
    "temperature":24.5,
    "wind_speed":7.7,
    "pressure":996.6
}
```

Также по адресу kirtrub.ru/docs можно открыть документацию swagger

3. Реализован базовый CI/CD пайплайн: когда в ветку main приходит push, Github Actions локально пересобирает и запускает проект на сервере.
